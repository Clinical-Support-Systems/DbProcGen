# Generated SQL (deterministic subtree)

This subtree is reserved for generated SQL artifacts.

- Paths and names must be deterministic.
- Files are generated from `specs/*.dbproc.json` and committed.
- Manual edits are not allowed.

## End-to-End Proof: GetUsersByFilter

This directory demonstrates a complete generation cycle for one procedure family:

### Generated Artifacts

1. **dbo_GetUsersByFilter.sql** - Wrapper procedure (public SQL API)
   - Routes to workers using concrete `IF/ELSE` branching based on FilterType and IsPaged
2. **dbo_GetUsersByFilter_name_paged.sql** - Worker for name-based paged search
   - Uses `OFFSET/FETCH` for efficient pagination
   - Queries against hand-authored `dbo.Users` table
3. **dbo_GetUsersByFilter_email_unpaged.sql** - Worker for email-based unpaged lookup
   - Direct equality match without paging overhead
   - Queries against hand-authored `dbo.Users` table
4. **generation-manifest.json** - Report showing which variants were emitted and why

### Design Highlights

- **Concrete wrapper routing:** The wrapper uses real `IF/ELSE` logic to dispatch based on parameter conditions
- **Meaningful worker differences:**
  - **Paged worker** uses `OFFSET/FETCH` to handle pagination efficiently
  - **Unpaged worker** omits paging logic entirely, optimized for single-result lookups
- **Hand-authored schema dependency:** Both workers reference `database/Schema/Tables/Users.sql`
- **ADR alignment:** Proves all ADRs 0001, 0002, 0004, 0005 in action

### Deterministic Properties

- **Naming:** `{schema}_{procedureName}.sql` for wrappers, `{schema}_{procedureName}_{workerSuffix}.sql` for workers
- **Ordering:** Specs sorted by LogicalName, routes sorted by WorkerSuffix
- **Headers:** All SQL files include `<auto-generated>` marker
- **Manifest:** JSON with stable structure, deterministic ordering, no timestamps

### ADR Compliance

This proof validates:
- **ADR 0001:** Build-time generation (not runtime); routing generated and visible in SQL
- **ADR 0002:** SQL project source-of-truth (files committed here); generated procedures depend on hand-authored schema
- **ADR 0004:** Wrapper + worker pattern (1 wrapper, 2 workers; wrapper includes concrete routing logic)
- **ADR 0005:** Deterministic artifacts (stable, reproducible, manifest report)
